# 소스·파일 공유 방법

`일경험` 폴더 전체(매뉴얼, `seed/`, Python 스크립트)를 여러 PC에서 **같은 내용**으로 쓰는 방법이다.

---

## 추천 (상황별)

| 상황 | 추천 | 이유 |
|------|------|------|
| **여러 PC, 인터넷 있음** | **GitHub 비공개 저장소** + `git clone` | 버전 고정, diff 확인, `.env`만 제외하기 쉬움 |
| **당일 빠르게 한 번** | **ZIP** + USB 또는 **OneDrive / Google Drive** 공유 링크 | 설치 없이 바로 복사 |
| **같은 공유기·사내망** | **공유 폴더** (SMB) | 대용량·방화벽 단순 |
| **코드만 가볍게** | GitHub **Release**에 zip 첨부 | clone 없이 다운로드 |

**한 줄 추천:** 장기·반복 → **GitHub(비공개)**. 당일 일회 → **ZIP + 클라우드 링크**.

---

## 1. GitHub 비공개 저장소 (가장 추천)

### 준비

1. [GitHub](https://github.com) 계정
2. New repository → **Private** → `weather-rp2040-lab-materials` 등
3. `일경험` 내용 push (아래 `.gitignore` 참고)

### 받는 쪽

```powershell
git clone https://github.com/사용자명/weather-rp2040-lab-materials.git
cd weather-rp2040-lab-materials
```

### `.gitignore` 예시 (저장소 루트)

```gitignore
.env
__pycache__/
.venv/
metrics_baseline.json
*.pyc
```

- **넣을 것:** 매뉴얼, `seed/`, `*.py`, `seed/power_realtime_seed.sql`
- **넣지 말 것:** `.env`(비밀번호·API 키), 개인 DB 덤프에 비밀 포함 시

### 배포 zip 대신

```powershell
git tag v2026-06-04
git push origin v2026-06-04
```

GitHub **Releases**에서 태그별 zip 다운로드 가능.

---

## 2. ZIP + 클라우드 (당일용)

### 만드는 쪽

```powershell
Compress-Archive -Path "C:\Users\user\일경험\*" -DestinationPath "$HOME\Desktop\ilgyeonghem.zip"
```

또는 탐색기에서 `일경험` 폴더 우클릭 → **압축**.

### 올리기

- **OneDrive / Google Drive:** zip 업로드 → **링크 공유** (보기 권한 있는 사람)
- **USB:** 회의실 PC에 직접 복사

### 받는 쪽

zip 풀기 → `일경험` 경로 기억 → 매뉴얼 5절·6절 경로를 **본인 PC 경로**로 수정.

---

## 3. 공유 폴더 (LAN)

```text
\\서버이름\share\일경험
```

- 수정 권한은 **한 명만** 두고, 나머지는 복사본을 `Documents`로 가져가는 편이 안전하다.
- `.env`는 공유 폴더에 두지 말고 각 PC에서 개별 생성.

## `.env` 전달

비밀번호·API 키는 zip/Git에 **넣지 않는다**.

- **예시만** 공유: 매뉴얼 7절 블록을 `env.example` 로 저장 (값은 `여기에_입력`)
- 각 PC에서 `.env` 를 직접 만들고 `MODBUS_PORT` / `MODBUS_MODE` 만 맞춘다.

```env
# env.example — 복사 후 .env 로 저장하고 값 채우기
WEATHER_SOURCE=openmeteo
MODBUS_MODE=rtu
MODBUS_PORT=COM3
...
```
